貝殼放大工程部徵夥伴.hr

貝殼放大是目前亞洲最大的一間群眾集資顧問公司，在 2014 年 10 月成立，至今共輔導成功超過 90 個專案，金額便已累積超過 4 億台幣。如果你對於群眾集資不是很瞭解，可以到群眾集資年報走過一遍歷史；也要知道歷史上首波群眾集資成功的案例是什麼；更多的更多，可以到群眾觀點看看。

Backer-Founder 不斷致力的目標是做好「-」，即贊助者與提案者之間的橋樑；中文的「貝殼」取自古代錢幣的用途，「放大」即傳達我們希望幫助團隊創造更多價值的核心理念。你也可以看看，我們的 CEO 林大涵在華人菁英論壇內怎麼說，裡面也有系列的影片就不多說。


顧問公司的技術團隊?

很多人會很好奇貝殼放大的技術團隊在做什麼，遇到很多面試者都是這樣說：
  • 就是幫忙提案者把網站做的美美的
  • 幫忙做一些金流的東西
  • 幫別人做集資平台
  • 做一些內部系統
  • 做數據分析
  • 幫忙解決一些網站問題
    • 其實都沒有說錯，這些都是我們的工作內容，但大家最不清楚的是，我們做了許多的「產品」！

    貝殼放大成立將近兩年，這段期間所累積的顧問經驗也好，內部、外部使用者的真（ㄨˊ）心（ㄌ一ˇ）回饋也好，都讓我們在滿足需求之餘，收斂與整合成目前的產品方向，主要有三大塊：獨立集資工具（FinTech）、大數據與 Marketing。


    獨立集資?

    撇除掉平台的限制與自製的困難度，要做到「集資」不外乎幾個點：金流、進度、故事。過去在平台只要有樣學樣，影片、文案、回饋定價行銷策略備齊，就能成為在平台上的一個集資專案。現在，「獨立集資」有許多工具解決金流，這個常是許多團隊沒辦法駕馭，且長期需要花費大量成本與 know-how 的部份。

    http://www.jarvish.com.tw/
    http://happystation.cc/
    http://www.patya.tw/
    http://www.luoproject.com/

    更棒的是，獨立集資做出的頁面，在未來集資完成後，都能繼續轉為形象官網，與自有長期販售的管道。在廣告當前，平台專案眾多未必能有許多曝光機會，相對更推薦獨立集資方式。人人都能做出屬於自己故事，與特色的頁面。

    如果說平台是拼圖，那獨立集資就是樂高，讓你說出有特色的集資故事！


    貝殼集器?

    Backme 貝殼集器是我們研發的第一個產品，也是目前所知，亞洲的第一個也是唯一的獨立集資工具，是在 2014/10/10 晚上 10:10，格外有意義的時間點由「台灣吧」打頭陣出發，也將在2016年第四季初做一個正式的產品發表。


    金流的彈性與多樣性

    簡單易上手，讓團隊不用自建網站開發團隊，更不用擔心金流安全性與穩定性。透過貝殼集器，只要一個「超連結」（即網址）就可完成串接。

    現在更為提升整體使用者體驗，還有推出嵌入式的贊助（付款）頁面，讓用戶不用經過跳轉，即可完成收款。

    近期我們也將推出「定期定額」、分期付款與國外金流等多樣的付款方式及多語系，不僅滿足更多類型的團隊與行銷手法需求，更可幫助更多台灣團隊面向國際，讓歐美團隊接觸亞洲市場！


    第三方套件

    以一個獨立「集資」工具來說，更重要的還是「如何讓使用者體會到集資的過程？」，盡可能的發揮自建網站的優勢，提昇每個專案的獨特之處。

    Backme 的進度嵌入，提供非常多客製化空間，任意控制欄位的顯示與進度條顏色，還有創新研發的進度條互動圖示，拉近與使用者的距離；鮮乳坊的牧場瓶數進度，是全由後台操作可以完成的，也是目前貝殼設計師對每個 Landing Page 發揮創意的地方。


    貝殼集器的下一步

    在接下來的半年到一年，我們會將貝殼集器正式產品發表，依舊會是最簡單易用的集資金流工具。此外，我們將著重在國際化，不單是語言切換，更是將台灣/亞洲產品帶到歐美，也是歐美產品帶進亞洲最好的工具橋樑。

    「行銷」便是接下來的主軸，結合貝殼放大過去輔導與執行數百件集資專案的經驗，會從最完整的廣告成效與轉換率追蹤出發，提供系統化的廣告投放建議，致力於創造客戶更大的成單量/金額成功為價值！

    其他的更多，歡迎有志的你，也來跟我們聊聊 FinTech 的將來。


    CrowdTrail

    這也是我們另外一項醞釀許久的產品，架構面來說是匯集全世界集資平台的所有專案數據；使用面來說，將會運用大數據分析與機器學習，提供集資專案執行上的建議。

    換個大家都知道的方式說，會是群眾集資界的 Google Analytics，亦或是 App Annie

    目前已累積破億筆資料，累計四大洲國家的平台共快20個，每小時正以破百萬筆的新紀錄成長中！

    對於大數據分析或機器學習等領域有興趣，面對大數據的儲存與效能的架構挑戰，這會是一個很好磨練的戰場，但更重要的是你得對群眾集資有想像。


    Backman

    最後，在內部自有工具，增加夥伴工作效率上我們也是不遺餘力的給予協助，貝殼人系統幫助大家 3 秒完成請假、10秒完成電子報帳，接下來還預計整合更多的第三方服務，製作內部專案管理系統，優化整體的顧問流程。

    透過這麼多的產品介紹，可以知道貝殼放大工程部的精實程度…從 FinTech、大數據分析到內部管理系統等都有機會接觸到，但有些人會問「那主要是什麼？」，以現階段來說重點即是「貝殼集器」走向 PMF，將集中精力在此發展；其餘產品則繼續玩 MVP 醞釀，以解決內部需求為主。

    後端工程師

    現階段開發業務內容會以貝殼集器為主，我們希望他除了有 Rails 的工作經驗外，還能有更多自己對於產品的想像，以及在他未來職涯目標上，貝殼放大能帶給他更多實質上的幫助。

    基本專業能力要求
    1. 使用 Ruby on Rails 開發/參與過至少一個專案
    2. Web 實務開發經驗二年以上 / 相關工作經驗一年以上
    3. 熟悉 HTML, CSS, Javascript
    4. 熱愛程式設計，喜歡學習新技術與高度重視程式開發品質。
    5. 具良好的團隊合作及溝通能力
    6. 具使用 git (github、bitbucket)之經驗
    7. 喜歡群眾集資的理念，熱愛貝殼放大的價值

    加分
    1. 具金流（智付寶/歐付寶）串接實務經驗
    2. 網頁爬蟲與數據分析相關作品
    3. 熟悉使用 RSpec
    4. 熟悉 AWS 架構以及完整佈署專案之能力
    5. Open Source 開發貢獻經驗
    6. 精通英文

    這是我們接觸使用者的一個方式

    Landing Page 幾乎是目前每個輔導專案的基本盤，從前的從前，也大概是公司成立初期，主要都為「顧問 - 設計 - 工程」三部曲的一個概念，從中隱藏了許多溝通成本造成的資訊落差與熱情消耗，不久便都交由設計直接使用 WIX 建立。

    目前一些線上作品如：
    http://www.patya.tw/
    http://www.talktotw.com/
    http://www.luoproject.com/

    而近期為提昇 Landing Page 的整體品質（不論使用者互動、流暢度還是廣告成效追蹤等），開始將需求高且複雜的獨立集資案件之頁面再拉回工程部門製作。除了滿足客戶需求之外，更重要的這是一個很好的修煉場，我們在上面逐漸完整化整套跨網域的廣告成效追蹤，以及 Google Analytics 與 Facebook Pixel 的轉換率強化；更因為這些 Landing Page 的差異性很大，有遊戲、設計品、科技品等，不同的頁面操作型態，讓我們多方嘗試不同的 Growth Hack 工具，從 UTM, CrazyEgg, Sumome, Optimizely, Keen.io, MixPanel, Customer.io, …，看似 routine 的事情，我們從中找到樂趣，進而內化與改進，反饋回到產品的設計開發上。
